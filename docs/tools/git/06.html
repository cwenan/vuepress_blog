# 概念

分支功能即从原项目中复制一个副本成为一条分支，在分支上进行开发，而不会影响到主分支的内容

1.Git 保存的不是文件的变化或者差异，而是一系列不同时刻的 快照

2.在进行提交操作时，Git 会保存一个提交对象（commit object）,该提交对象会包含一个指向暂存内容快照的指针。 但不仅仅是这样，该提交对象还包含了作者的姓名和邮箱、提交时输入的信息以及指向它的父对象的指针。

三个对象：

  • blob对象：用来保存单个文件的快照
  • 树对象：包含目录结构和blob对象索引
  • 提交对象：包含树对象的指针和所有提交信息

首次提交产生的提交对象没有父对象，普通提交操作产生的提交对象有一个父对象， 而由多个分支合并产生的提交对象有多个父对象。

Git默认的分支为master，即git init后自动创建的分支，它与其他分支完全没用区别。在多次进行提交操作后，master分支的指针会自动向前移动，指向最后最新提交的那个提交对象(A-->B-->C)

# 原有一个master分支，当前提交对象上创建一个testing分支

# 合并分支

背景：

  • 你在开发一个项目，为了完成一个新功能，创建了一条分支，并且在这个分支上面进行开发
  • 突然，boss打电话过来说线上项目出bug了，需要紧急修复。

这是原来的master分支

使用git checkout -b iss53创建一个新的分支来开发那个新功能。

你有了一定的进展，并且在该分支上做了一些提交。

现在，你接到电话，要去修复bug，需要的仅仅是切换回原理的master分支上git checkout master

这里在切换分支之前，尽可能保留干净的状态，即坚持当前的工作区和暂存区是否还有没有被提交的修改，可能会发生冲突，使用 git status来检查

为了修复这个bug，你创建了一个hotfix分支git checkout -b hotfix,并且把bug修复了

这时你需要把hotfix分支合并到主分支master上来

# 先切换到你想要被合并的分支上
$ git checkout master

# 执行合并操作,合并对应的分支
$ git merge hotfix

看到git给出的提示：fast-forward(快进)，因为由于你想要合并的分支 hotfix 所指向的提交 C4 是你所在的提交 C2 的直接后继， 因此 Git 会直接将指针向前移动。换句话说，当你试图合并两个分支时， 如果顺着一个分支走下去能够到达另一个分支，那么 Git 在合并两者的时候， 只会简单的将指针向前推进（指针右移），因为这种情况下的合并操作没有需要解决的分歧——这就叫做 “快进（fast-forward）”。

现在的bug已经修复

问题修复后，应先把hotfix分支删除掉，git branch -d hotfix，因为不需要它了，并且使用git checkout iss53继续进行开发新功能

假如此时你已经把新功能开发好了，并且把iss53分支合并到master分支上面来

这和你之前合并 hotfix 分支的时候看起来有一点不一样。 在这种情况下，你的开发历史从一个更早的地方开始分叉开来（diverged）。 因为，master 分支所在提交并不是 iss53 分支所在提交的直接祖先，Git 不得不做一些额外的工作。 出现这种情况的时候，Git 会使用两个分支的末端所指的快照（C4 和 C5）以及这两个分支的公共祖先（C2），做一个简单的三方合并。

合并提交

和之前将分支指针向前推进所不同的是，Git 将此次三方合并的结果做了一个新的快照并且自动创建一个新的提交指向它。 这个被称作一次合并提交，它的特别之处在于他有不止一个父提交。

这时你已经完成了任务，把iss53分支删除即可git branch -d iss53

冲突

使用git merge branch_name时，常常会出现冲突

当你在两个不同分支中，对同一个文件的同一个位置进行了不同的修改，Git无法判断如何合并，因此产生了冲突，可以使用git status命令来查看那些因为包含冲突而处于未合并状态的文件

  • 由等线分割成两部分
  • 等线上半部分，也就是HEAD指向的部分，是当前分支所修改的内容
  • 等线下部分，也就是iss53所对应的部分，是iss53分支所修改的内容
  • 因为git无法做出判断使用哪一部分，所以由人来手动删除

解决冲突：

  • 选择使用等线分割的两部分中的一个，或者自行合并这些内容
  • 把<<<< >>>>> =====这些符号删除掉
  • 当解决所有冲突后，使用git add命令来将其标记为冲突已解决
  • 使用git status命令来查看冲突是否已被解决
  • 使用git commit来保存新的版本(可选)
