# 打标签

在GitHub里面，我们常常可以看到一些关于项目版本发步的标签，来突出重要性

比如vue项目里面的各种tags

创建标签

Git 支持两种标签：轻量标签（lightweight）与附注标签（annotated）

轻量标签很像一个不会改变的分支——它只是某个特定提交的引用。

而附注标签是存储在 Git 数据库中的一个完整对象， 它们是可以被校验的，其中包含打标签者的名字、电子邮件地址、日期时间， 此外还有一个标签信息，并且可以使用 GNU Privacy Guard （GPG）签名并验证。

通常会建议创建附注标签，这样你可以拥有以上所有信息。但是如果你只是想用一个临时的标签， 或者因为某些原因不想要保存这些信息，那么也可以用轻量标签。

附注标签

# 创建标签，-m后面填写标签相关信息,比如创建一个v1.0的标签
$ git tag -a v1.0 -m"comment"

# 查看某个标签的具体情况,比如查看v1.0
$ git show v1.0

推送标签

git push命令并不会把标签推送到远程仓库里，创建完标签，需要手动推送标签

# 手动推送
$ git push origin tag_name

# 一次性推送很多标签
$ git push origin --tags

删除标签

# 删除本地标签
$ git tag -d  tag_name

# 删除远程标签
$ git push origin --delete tag_name
